# 説明資料（e-Stat + 友好度グルーピング React）

## 目的
e-Stat（ver3.0）のデータ取得により首都圏の「総人口」を都県（横軸）×西暦（縦軸、降順）で表形式表示する。あわせて、与えられた友好度CSVをもとに都道府県を最大3グループに最適化し、人口表をグループごとに色分けして可視化する。

## 実装で参照したサイトと記述内容
1. [e-Stat API 公式（トップ）](https://www.e-stat.go.jp/api/)
   - API利用登録（無料）と `appId` の発行
   - APIバージョン3.0の概要
2. [e-Stat API仕様書（ver3.0 manual）](https://www.e-stat.go.jp/api/api-info/e-stat-manual3-0)
   - `statsDataId` を指定して統計表データを取得するためのエンドポイント設計
   - リクエストパラメータ（メタ取得フラグ等）
3. [e-Stat API Viewer](https://mtoyokura.github.io/estat-api-viewer/)
   - キーワード検索で対象の統計表を見つけ、`statsDataId` を特定する用途

## 「どのように検索して該当記事を見つけたか」
統計表情報取得の段階で、e-Stat API Viewer / 仕様書を次のキーワード方針で探索した。
- 「全国総人口に占める人口割合」などの文言検索
- 「都道府県別」「総人口」「年次」「人口推計」等の組み合わせ
該当する統計表を見つけた後、Viewerに表示される `statsDataId` を控え、React側の `statsDataId` 入力に反映した。

## 参考プログラムのどの部分を変更し、どの部分に適用したか
本実装では、友好度CSV・最適グループ化・色分け表示は課題要件に合わせて自前実装している。
主な実装場所は以下。

1. 友好度CSV解析（`-`/数値の解釈、片側にだけ数値がある場合の対称化）
   - `src/lib/friendship.ts`
2. 「同一グループ内の友好度合計」を最大化するグループ探索（最大3グループ）
   - `src/lib/grouping.ts`
   - 都道府県を再帰的に最大3グループへ割り当て、グループ内ペアのスコア合計を評価
3. 色分けした人口表（都県を列、年を行）
   - `src/components/PopulationTable.tsx`
4. e-Stat（ver3.0）ライブ取得（実装は“最小のベストエフォート”）
   - `src/lib/estat.ts`
   - `getStatsData` を呼び出し、レスポンス中から `{cdArea, time, value}` に近い要素を探索して `PopulationMatrix` へ変換する方針
   - 統計表によってレスポンス形式が異なるため、統計表ごとにパース部分の調整余地がある

## 使い方（画面での操作）
1. e-Stat API の `appId` と対象の `statsDataId` を入力し「ライブ取得」を実行する
2. 友好度CSVをアップロードする（ファイル形式はヘッダ付きの対称行列を想定）
3. 表が自動で色分けされ、各グループの所属都県と最大友好度合計が表示される
